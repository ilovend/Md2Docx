# 并行任务清单 (Parallel Backlog)

> **用途**：当等待 AI 响应、代码评审或构建时，可执行的低切换成本任务  
> **最后更新**：2026-01-26

---

## 📋 可并行任务分类

### 1. 文档维护 (低风险，随时可做)

- [ ] 检查 `docs/index.md` 中的链接有效性
- [ ] 补充缺失的代码注释
- [ ] 更新 CHANGELOG.md
- [ ] 整理 ADR 编号连续性
- [ ] 检查 Prompt 模板字段完整性
- [ ] 更新 README 中的功能描述

### 2. 测试补充 (中等优先级)

- [ ] 为核心函数补充单元测试
- [ ] 为 API 端点补充集成测试
- [ ] 更新测试用例文档
- [ ] 检查测试覆盖率报告

### 3. 代码清理 (低优先级)

- [ ] 删除未使用的导入
- [ ] 统一代码格式化
- [ ] 修复 Lint 警告
- [ ] 优化导入顺序
- [ ] 删除注释掉的代码

### 4. ADR 整理

- [ ] 检查所有 ADR 状态是否准确
- [ ] 补充缺失的 ADR（技术选型决策）
- [ ] 更新已过时的 ADR 状态为"已取代"

### 5. 依赖维护

- [ ] 检查依赖更新 (`npm outdated` / `pip list --outdated`)
- [ ] 评估是否需要升级
- [ ] 更新 package-lock.json / requirements.txt

## ⏰ 任务时间估计

| 任务类型 | 平均耗时 | 适合场景 |
|:---|:---|:---|
| 链接检查 | 5-10 分钟 | 等待构建 |
| 补充注释 | 10-20 分钟 | 等待评审 |
| 单元测试 | 20-30 分钟 | 等待 AI 响应 |
| 代码格式化 | 5 分钟 | 快速切换 |
| ADR 整理 | 15-30 分钟 | 长时间等待 |

## 📌 使用提示

1. **选择任务**：根据等待时间选择合适耗时的任务
2. **避免深度工作**：选择不需要深度思考的任务
3. **记录完成**：完成后移动到"已完成任务记录"
4. **定期清理**：每周清理已完成记录，保留近一个月
